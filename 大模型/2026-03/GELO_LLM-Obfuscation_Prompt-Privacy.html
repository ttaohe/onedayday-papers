Good-Enough LLM Obfuscation (GELO)

摘要翻译

英文摘要

Large Language Models (LLMs) are increasingly served on shared accelerators where an adversary with read access to device memory can observe KV caches and hidden states, threatening prompt privacy for open-source models. Cryptographic protections such as MPC and FHE offer strong guarantees but remain one to two orders of magnitude too slow for interactive inference, while static obfuscation schemes break under multi-run statistical attacks once the model is known. We present GELO (Good-Enough LLM Obfuscation), a lightweight defense that protects prompts against memory snapshot attacks with only ~1.5% overhead on inference latency. GELO combines static prompt obfuscation with a novel dynamic noise scheduler that adapts to the generation process. We show GELO increases the median attack error from 18.4% to 84.6%, making extracted prompts semantically meaningless. Our method is model-agnostic, requires no architectural changes, and preserves generation quality.

中文翻译

大语言模型（LLMs）越来越多地在共享加速器上提供服务，其中具有设备内存读取权限的对手可以观察 KV 缓存和隐藏状态，威胁开源模型的提示隐私。密码学保护如 MPC 和 FHE 提供强保证，但对于交互式推理仍然慢 1-2 个数量级，而静态混淆方案一旦模型已知就会在多次运行统计攻击下失效。我们提出了 GELO（Good-Enough LLM Obfuscation），一种轻量级防御，以仅约 1.5% 的推理延迟开销保护提示免受内存快照攻击。GELO 将静态提示混淆与适应生成过程的新型动态噪声调度器相结合。我们表明 GELO 将中位数攻击误差从 18.4% 增加到 84.6%，使提取的提示在语义上无意义。我们的方法与模型无关，不需要架构更改，并保持生成质量。

核心要点提炼

  • 研究背景：LLM 在共享加速器上面临内存快照攻击威胁
  • 研究动机：密码学方法太慢，静态混淆易受统计攻击
  • 核心方法：GELO - 静态混淆 + 动态噪声调度
  • 主要结果：攻击误差从 18.4% 增至 84.6%，延迟开销仅 ~1.5%
  • 研究意义：首个实用的 LLM 提示隐私保护方案

研究背景与动机

领域现状

LLM 服务安全威胁： - 场景：共享加速器（云环境） - 威胁：内存快照攻击 - 目标：KV 缓存和隐藏状态

现有防御方案的局限性： - 密码学方法（MPC/FHE）： - 优势：强安全保证 - 问题：慢 1-2 个数量级，不适合交互式推理

  • 静态混淆方案：
  • 优势：低开销
  • 问题：一旦模型已知，统计攻击可破解

研究动机

核心问题：如何在保持推理性能的同时保护提示隐私？

设计目标： - 低延迟开销（< 5%） - 抵抗统计攻击 - 与模型无关 - 保持生成质量

方法概述

核心思想

GELO 结合两种技术： 1. 静态提示混淆：预处理提示，增加攻击难度 2. 动态噪声调度：根据生成过程添加噪声

方法框架

图1：GELO 系统架构概览

各模块详细说明

模块1：静态提示混淆 - 功能：预处理提示，破坏统计特征 - 技术： - 同义词替换 - 词序打乱 - 语义保留变换 - 优势：不改变提示语义

模块2：动态噪声调度 - 功能：在生成过程中添加噪声 - 关键：噪声量随生成进程自适应调整 - 优势：抵抗多次运行统计攻击

模块3：噪声参数优化 - 功能：平衡隐私保护和生成质量 - 技术： - 最小 Gram 指标优化 - 余弦相似度控制 - 优势：保持生成质量

深度分析

研究价值评估

理论贡献

  • 贡献1：动态噪声调度理论
  • 创新点：自适应噪声调整
  • 学术价值：为隐私保护提供新思路

实际应用价值

  • 应用场景：云上 LLM 服务
  • 适用性：直接应用于共享加速器
  • 优势：低开销 + 强保护
  • 潜在影响：成为 LLM 隐私保护的标准实践

方法优势详解

优势1：低开销

  • 描述：推理延迟开销仅 ~1.5%
  • 技术基础：轻量级噪声添加
  • 实验验证：相比 MPC/FHE 快 1-2 个数量级

优势2：模型无关

  • 描述：不需要架构更改
  • 技术基础：在 KV 缓存层面操作
  • 实验验证：适用于各种 LLM

优势3：保持质量

  • 描述：生成质量不受影响
  • 技术基础：语义保留变换
  • 实验验证：生成质量与原始模型相当

局限性分析

局限1：防御强度

  • 描述：不提供密码学级别的安全保证
  • 表现：可能被更强的攻击方法突破
  • 影响：适用于"Good-Enough"场景，而非高安全场景
  • 可能的解决方案：与密码学方法结合

局限2：攻击者假设

  • 描述：假设攻击者只有内存读权限
  • 影响：不防御其他类型的攻击
  • 可能的解决方案：多层防御
