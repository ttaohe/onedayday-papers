Hardware-Software Co-design for 3D-DRAM-based LLM Serving Accelerator

Cong Li, Yihan Yin, Chenhao Xue, Zhao Wang, Fujun Bai, Yixin Guo, Xiping Jiang, Qiang Wu, Yuan Xie, Guangyu Sun 8.7/10 arXiv PDF

摘要

大语言模型（LLM）已被广泛用于在线生成服务，其中大量 LLM 实例共同处理请求到达率波动和请求长度变化的负载。为了高效执行共存的计算密集型和内存密集型算子，基于近内存处理（NMP）的计算范式已被广泛提出。然而，现有的 NMP 设计采用粗粒度的 KV cache 管理和不灵活的 attention 执行流程。这些限制阻碍了这些提案高效处理高度动态的 LLM serving 工作负载，限制了它们加速 LLM serving 的能力。

为了解决这些问题，我们提出了 Helios，一个基于混合键合的 LLM Serving 加速器。Helios 旨在弥合 LLM serving 中 KV cache 管理的动态特性与 NMP 处理引擎（PE）之间分布式、非均匀内存抽象之间的根本差距。为此，我们设计了用于分布式分块 attention 执行的 PE 内执行流程和 PE 间通信原语。我们进一步提出了空间感知的 KV cache 分配机制，以平衡 attention 工作负载分布，同时最小化 PE 间数据传输开销。与现有 GPU/NMP 设计相比，Helios 实现了 3.25 倍（几何平均）加速和 3.36 倍（几何平均）更好的能效，P50/P99 token 间时间延迟最多降低 72%/76%。

原文

Large language models (LLMs) have been widely deployed for online generative services, where numerous LLM instances jointly handle workloads with fluctuating request arrival rates and variable request lengths. To efficiently execute coexisting compute-intensive and memory-intensive operators, near-memory processing (NMP) based computing paradigm has been extensively proposed. However, existing NMP designs adopt coarse-grained KV cache management and inflexible attention execution flow. Such limitations hinder these proposals from efficiently handling highly dynamic LLM serving workloads, limiting their ability to accelerate LLM serving. To tackle these problems, we propose Helios, a Hybrid-bonding-based LLM Serving accelerator. Helios aims to bridge the fundamental gap between the dynamic nature of KV cache management in LLM serving and the distributed, non-uniform memory abstraction among NMP processing engines (PEs). To this end, we design both the intra-PE execution flow and the inter-PE communication primitives for distributed tiled attention execution. We further propose spatially-aware KV cache allocation mechanism to balance the attention workload distribution while minimizing the inter-PE data transfer overhead. Compared with existing GPU/NMP designs, Helios achieves 3.25 times (geomean) speedup and 3.36 times (geomean) better energy efficiency, along with up to 72%/76% P50/P99 time-between-tokens degradation.

核心要点

  • 研究背景：LLM serving 面临高度动态工作负载，需要更高效的硬件支持
  • 研究动机：现有 NMP 设计的 KV cache 管理和 attention 执行不够灵活
  • 核心方法：Helios - 混合键合 3D-DRAM LLM serving 加速器
  • 主要结果：3.25x 加速，3.36x 能效提升，P50/P99 延迟降低 72%/76%
  • 研究意义：首个针对 LLM serving 的 3D-DRAM 硬件协同设计

研究背景与动机

领域现状

LLM Serving 的挑战： - 请求到达率波动 - 请求长度变化 - 计算/内存密集型算子共存

Near-Memory Processing (NMP)： - 将计算移到内存附近，减少数据传输 - 但现有设计采用粗粒度 KV cache 管理 - Attention 执行流程不够灵活

研究动机

核心问题：如何设计高效的 3D-DRAM LLM serving 加速器？

创新点与方法

核心思想

Helios 是一个基于混合键合（Hybrid-bonding）的 3D-DRAM LLM serving 加速器，通过软硬件协同设计解决以下问题： 1. PE 内执行流程优化 2. PE 间通信原语 3. 空间感知的 KV cache 分配

方法框架

图1：Helios 系统架构概览

各模块详细说明

模块1：混合键合技术 - 功能：实现高性能 3D-DRAM 集成 - 技术：Hybrid bonding 连接不同类型的 DRAM - 优势：平衡性能、功耗和成本

模块2：分布式 Tiled Attention - 功能：跨 PE 的分块 attention 执行 - 关键：空间感知的 KV cache 分配 - 优势：最小化 PE 间数据传输

模块3：KV Cache 管理优化 - 功能：细粒度 KV cache 管理 - 技术：动态 KV cache 分配和迁移 - 优势：适应动态工作负载

实验结果

对比指标 提升倍数
加速比（几何平均） 3.25x
能效提升 3.36x
P50 延迟改善 72%
P99 延迟改善 76%

结果分析： - 与现有 GPU/NMP 设计相比显著提升 - 空间感知 KV cache 分配有效 - 混合键合技术实现高性能集成
