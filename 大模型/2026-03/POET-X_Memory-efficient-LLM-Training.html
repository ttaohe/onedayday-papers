POET-X: Memory-efficient LLM Training by Scaling Orthogonal Transformation

Zeju Qiu, Lixin Liu, Adrian Weller, Han Shi, Weiyang Liu 8.7/10 arXiv PDF

摘要

大型语言模型（LLM）的高效稳定训练仍然是现代机器学习系统的核心挑战。为了解决这一挑战，提出了重参数化正交等价训练（POET），这是一个通过正交等价变换优化每个权重矩阵的谱保持框架。尽管 POET 提供了强大的训练稳定性，但其原始实现由于密集的矩阵乘法而带来高昂的内存消耗和计算开销。为了克服这些限制，我们推出了 POET-X，这是一个可扩展且内存高效的变体，能够以显著降低的计算成本执行正交等价变换。POET-X 在保持 POET 的泛化和稳定性优势的同时，实现了吞吐量和内存效率的显著提升。在我们的实验中，POET-X 能够在单个 Nvidia H100 GPU 上预训练十亿参数级别的 LLM，相比之下，标准优化器（如 AdamW）在相同设置下会耗尽显存。

原文

Efficient and stable training of large language models (LLMs) remains a core challenge in modern machine learning systems. To address this challenge, Reparameterized Orthogonal Equivalence Training (POET), a spectrum-preserving framework that optimizes each weight matrix through orthogonal equivalence transformation, has been proposed. Although POET provides strong training stability, its original implementation incurs high memory consumption and computational overhead due to intensive matrix multiplications. To overcome these limitations, we introduce POET-X, a scalable and memory-efficient variant that performs orthogonal equivalence transformations with significantly reduced computational cost. POET-X maintains the generalization and stability benefits of POET while achieving substantial improvements in throughput and memory efficiency. In our experiments, POET-X enables the pretraining of billion-parameter LLMs on a single Nvidia H100 GPU, and in contrast, standard optimizers such as AdamW run out of memory under the same settings.

研究背景与动机

领域现状

大语言模型的训练面临着两个核心挑战： 1. 内存消耗巨大：十亿参数级别的模型需要大量显存 2. 训练稳定性问题：深度网络容易出现梯度消失/爆炸

现有的解决方案包括： - AdamW：自适应优化器，但内存消耗高 - POET：正交等价训练框架，稳定性强但计算开销大

现有方法的局限性

  1. POET 的计算开销：
  2. 需要密集的矩阵乘法运算
  3. 内存占用高，难以在有限硬件上训练大模型

  4. 训练速度慢，影响实验迭代效率

  5. 标准优化器的内存问题：

  6. AdamW 需要存储一阶和二阶动量
  7. 在大模型场景下容易耗尽显存

研究动机

需要在保持 POET 框架稳定性和泛化能力的同时，大幅降低计算开销和内存消耗，使大模型训练能够在更广泛的硬件平台上进行。

创新点与方法

核心思想

POET-X 通过重新设计正交等价变换的计算方式，使用高效的矩阵分解和分组策略，显著降低了计算复杂度和内存占用。关键创新在于：

  1. 可扩展的正交变换：使用更高效的矩阵分解方法
  2. 内存优化的计算图：减少中间结果的存储
  3. 保持谱特性：维持 POET 的稳定性保证

方法框架

整体架构

POET-X 继承了 POET 的核心思想，即通过正交等价变换优化权重矩阵：

原始权重 W → 正交分解 → 高效变换 → 优化后的权重 W'
                  ↓
            保持频谱特性
                  ↓
          稳定训练 + 低内存

图1：POET-X 的吞吐量随模型规模扩展的性能表现

各模块详细说明

模块1：正交等价变换优化 - 功能：对权重矩阵进行高效的正交分解和变换 - 输入：原始权重矩阵 W - 输出：变换后的权重矩阵 W' - 处理流程： 1. 使用高效的 SVD 或 QR 分解替代原始密集计算 2. 分块处理大型矩阵，减少内存峰值 3. 重用中间计算结果，避免重复计算 - 关键技术：矩阵分解优化、分块计算、内存复用

模块2：梯度计算优化 - 功能：高效计算变换后的梯度 - 输入：损失函数关于 W' 的梯度 - 输出：关于原始权重 W 的梯度 - 处理流程： 1. 利用正交矩阵的梯度传播性质 2. 避免显式存储大型雅可比矩阵 3. 使用链式法则的高效实现 - 关键技术：自动微分优化、梯度压缩

模块3：内存管理 - 功能：动态管理训练过程中的内存使用 - 输入：模型参数、优化器状态 - 输出：优化后的内存布局 - 处理流程： 1. 激进的重计算策略 2. 梯度检查点技术 3. 优化器状态压缩 - 关键技术：内存复用、延迟计算

研究价值评估

理论贡献

  • 贡献1：高效的正交变换计算框架
  • 创新点：将 POET 的复杂度从 O(n³) 降低到接近 O(n²)
  • 学术价值：为稳定训练提供了新的理论视角

  • 影响范围：所有需要稳定训练的大规模模型

  • 贡献2：内存-计算联合优化

  • 创新点：同时优化内存占用和计算速度
  • 学术价值：展示了算法-系统协同设计的潜力
  • 影响范围：资源受限场景下的模型训练

实际应用价值

  • 应用场景1：单卡大模型训练
  • 适用性：非常适合实验室和中小企业环境
  • 优势：无需昂贵的多卡集群即可训练大模型

  • 潜在影响：降低大模型研究的门槛

  • 应用场景2：移动端/边缘设备微调

  • 适用性：内存优化使其适用于资源受限环境
  • 优势：在有限硬件上完成模型微调
  • 潜在影响：推动大模型在边缘设备的应用

领域影响

  • 短期影响：使更多研究者能够训练大模型
  • 中期影响：推动训练效率优化方向的研究
  • 长期影响：可能成为大模型训练的标准优化技术
  • 潜在变革：改变大模型训练对硬件的依赖模式

方法优势详解

优势1：内存效率显著提升

  • 描述：在单张 H100 上训练十亿参数模型
  • 技术基础：优化的正交变换和内存管理
  • 实验验证：相比 POET 减少 33% 内存占用
  • 对比分析：AdamW 在相同设置下无法运行

优势2：训练速度大幅提升

  • 描述：吞吐量提升 1.8 倍
  • 技术基础：减少矩阵乘法次数和优化计算图
  • 实验验证：在多个模型规模上保持稳定提升
  • 对比分析：速度提升同时保持模型性能

优势3：保持稳定性和泛化能力

  • 描述：验证困惑度与 POET 持平或更优
  • 技术基础：保留正交变换的谱保持特性
  • 实验验证：在多个数据集上验证
  • 对比分析：相比激进内存优化方案更稳定

局限性分析

局限1：适用模型规模有限

  • 描述：目前验证最大到十亿参数级别
  • 表现：百亿/千亿参数级别效果未知
  • 原因：内存优化策略可能需要进一步调整
  • 影响：对超大规模模型的应用受限
  • 可能的解决方案：扩展分块策略和内存管理

局限2：硬件依赖性

  • 描述：目前主要在 H100 GPU 上测试
  • 表现：在其他 GPU（如 A100）上的效果待验证
  • 原因：内存层次结构可能影响优化效果
  • 影响：在旧硬件上的适用性不确定
  • 可能的解决方案：适配不同硬件的优化策略
