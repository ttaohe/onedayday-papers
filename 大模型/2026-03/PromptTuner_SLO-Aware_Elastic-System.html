摘要翻译

英文摘要

Prompt tuning has become a prominent strategy for enhancing the performance of Large Language Models (LLMs) on downstream tasks. Many IT enterprises now offer Prompt-Tuning-as-a-Service to fulfill the growing demand for prompt tuning LLMs on downstream tasks. Their primary objective is to satisfy users' Service Level Objectives (SLOs) while reducing resource provisioning costs. Nevertheless, our characterization analysis for existing deep learning resource management systems reveals that they are insufficient to optimize these objectives for LLM prompt tuning workloads.

In this paper, we introduce PromptTuner, an SLO-aware elastic system to optimize LLM prompt tuning. It contains two innovations. (1) We design a Prompt Bank to identify efficient initial prompts to expedite the convergence of prompt tuning. (2) We develop a Workload Scheduler to enable fast resource allocation to reduce the SLO violation and resource costs. In our evaluation, PromptTuner reduces SLO violations by 4.0x and 7.9x, and lowers costs by 1.6x and 4.5x, compared to INFless and ElasticFlow respectively.

中文翻译

Prompt tuning 已成为增强大语言模型（LLM）在下游任务上性能的主要策略。许多 IT 企业现在提供 Prompt-Tuning-as-a-Service 来满足对 LLM 进行 prompt tuning 的下游任务需求。他们的主要目标是在满足用户服务级别目标（SLO）的同时降低资源配置成本。然而，我们对现有深度学习资源管理系统的表征分析表明，它们不足以优化 LLM prompt tuning 工作负载的这些目标。

在本文中，我们介绍了 PromptTuner，一个 SLO 感知的弹性系统来优化 LLM prompt tuning。它包含两个创新：（1）我们设计了一个 Prompt Bank 来识别高效的初始 prompt，以加速 prompt tuning 的收敛。（2）我们开发了一个 Workload Scheduler 来实现快速的资源分配，以减少 SLO 违规和资源成本。在我们的评估中，与 INFless 和 ElasticFlow 相比，PromptTuner 分别将 SLO 违规减少了 4.0 倍和 7.9 倍，成本降低了 1.6 倍和 4.5 倍。

研究背景与动机

领域现状

Prompt Tuning 作为服务： - IT 企业提供 Prompt-Tuning-as-a-Service - 用户需求：满足 SLO（Service Level Objectives） - 服务商目标：降低资源成本

现有系统的局限性： - 现有深度学习资源管理系统（如 INFless、ElasticFlow）对 LLM prompt tuning 工作负载优化不足 - LLM prompt tuning 有独特的特性： - 收敛速度高度依赖初始 prompt 质量 - 资源需求随训练进程动态变化 - 不同任务的 SLO 要求不同

研究动机

核心问题：如何在满足 SLO 的前提下，优化 LLM prompt tuning 的资源分配？

研究问题

核心研究问题

设计一个 SLO 感知的弹性系统，用于优化 LLM prompt tuning： 1. 如何识别高效的初始 prompt 以加速收敛？ 2. 如何快速分配资源以减少 SLO 违规和资源成本？

方法概述

核心思想

PromptTuner 是一个针对 LLM prompt tuning 的 SLO 感知弹性系统，包含两个核心创新： 1. Prompt Bank：识别高效初始 prompt 2. Workload Scheduler：快速资源分配

方法框架

图1：PromptTuner 系统架构，展示 Prompt Bank 和 Workload Scheduler 两个核心组件

各模块详细说明

模块1：Prompt Bank - 功能：识别和存储高效的初始 prompt - 输入：历史 prompt tuning 任务 - 输出：高效的初始 prompt 候选 - 核心思想： - 不同任务的 prompt tuning 收敛速度差异很大 - 好的初始 prompt 可以大幅减少收敛时间 - 通过学习和历史记录，识别出适合不同任务类型的初始 prompt - 关键技术： - Prompt 相似度度量 - 收敛速度预测模型 - Prompt 索引和检索

模块2：Workload Scheduler - 功能：快速资源分配，减少 SLO 违规 - 输入：任务队列、SLO 要求、资源状态 - 输出：资源分配决策 - 核心思想： - LLM prompt tuning 的资源需求随训练进程动态变化 - 需要快速响应资源需求变化 - 在 SLO 约束下优化资源利用率 - 关键技术： - 弹性资源分配算法 - SLO 感知的任务调度 - 资源预留和释放策略
